# AGENT_SMITH -- Patch Notes v7 (Text + TTS + ZIP output)

## Проблема
Агент возвращал только ZIP-архив без текстового ответа в чат.
Пользователь не видел что именно сделал агент без скачивания архива.

## Исправление: _run_code_pipeline

Новая последовательность после выполнения задачи:

  1. TEXT  -- Читаем result.md или output.txt из артефактов
              Отправляем текст в чат (до 6 сообщений по 3800 символов)
              Формат: "📄 Ответ агента:\n\n<текст>"

  2. TTS   -- Если пользователь имеет право 'tts':
              Берём первые 900 символов ответа
              Убираем markdown (* _ ` # < >)
              Синтезируем MP3 через tts_engine.synthesize()
              Отправляем аудио: "🎙 Озвучка ответа"

  3. ZIP   -- Отправляем архив: "📦 Архив (код + вывод + файлы) N KB"
              Fallback: отдельные файлы если ZIP не отправился

  4. STATUS -- "✅ Готово  ·  3 файлов · 📦 ZIP · 🎙 TTS"
               Показывает что именно было отправлено

## Логика выбора текста для вывода

Приоритет артефактов:
  1. result.md  -- общий LLM-ответ (analyze, общие задачи)
  2. output.txt -- stdout/stderr выполнения кода

Минимальная длина: 20 символов (пустые файлы пропускаются)
TTS макс: 900 символов (ограничение качества синтеза)
Текст в чат макс: 6 сообщений × 3800 символов = ~23KB

## Примечания

- TTS проверяет право 'tts' по роли пользователя
- TTS не прерывает флоу при ошибке (try/except pass)
- Если result.md не найден -- текст в чат не отправляется
  (для задач где агент только генерирует файлы без текстового ответа)
